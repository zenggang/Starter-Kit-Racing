# Track Editor Mobile Console Design

## Context

- Current surface:
  - `src/components/TrackEditorClient.tsx` 当前将赛道名称、工具按钮、3D 编辑区、操作提示和已保存赛道列表放在同一个桌面优先布局里。
  - `src/app/globals.css` 目前只给赛道编辑器提供了基础单栏折叠：窄屏时 `track-editor-grid` 直接改成单列，地图区仍按普通内容块向下堆叠。
  - `js/TrackEditorRuntime.js` 已经具备地图编辑所需的核心交互：单指或鼠标拖拽画路、擦除、平移、缩放、终点方向切换。当前问题不是编辑能力缺失，而是移动端页面壳层没有围绕这些交互做一屏布局。
- Why this design is needed:
  - 用户已经明确指出“自定义地图完全没考虑移动端布局”，目标不是给现有桌面布局做被动缩放，而是重排成横屏移动端可直接操作的一屏控制台。
  - 用户进一步确认：移动端只保留编辑操作同屏；“我的赛道/已保存列表”不与地图编辑器常驻同屏；A 方案的双侧窄控制台 + 中央固定地图台方向成立，但操作按钮还需要比第一版 mockup 更小、更集中。

## Goal

- Primary outcome:
  - 把自定义赛道页重排成移动端横屏优先的掌机式控制台：中央固定比例地图台、左右窄控制区、一屏内完成核心编辑动作。
- User-visible result:
  - 在移动端横屏进入 `/track-editor` 时，玩家无需滚动即可看到并操作地图编辑器。
  - 高频操作按钮更紧凑，不会像主行动按钮那样过度占据空间。
  - 地图区保持居中、固定大小和清晰边界，支持继续使用现有拖拽、平移、缩放交互。

## Scope

- In scope:
  - 调整 `TrackEditorClient` 的 DOM 组织，使移动端存在明确的“工具区 / 地图区 / 状态与保存区”分组。
  - 调整 `globals.css` 中赛道编辑器相关布局、尺寸、间距和断点规则。
  - 在移动端把“已保存赛道列表”移出主编辑同屏区域，改成非同屏入口或次级面板。
  - 保持 `TrackEditorRuntime` 的编辑交互语义不变，但要确保页面容器对拖拽地图这种交互友好。
  - 补充或更新组件测试，确保移动端专用入口和主编辑动作未回归。
- Out of scope:
  - 不修改赛道存储 API、地图合法性校验、保存语义、删除语义或赛道数据结构。
  - 不重写 `js/TrackEditorRuntime.js` 的 3D 编辑算法。
  - 不引入新的地图编辑功能，例如撤销/重做、多图层、分享、预设模板。
  - 不扩展到大厅、房间页或比赛页的新一轮移动端改版。

## Constraints

1. 修改范围必须尽量小，优先复用现有组件、现有按钮语义和现有状态。
2. 核心编辑动作必须在移动端一屏内完成：
   - 画路
   - 擦除
   - 新建
   - 保存
   - 赛道名称编辑
3. “我的赛道/已保存列表”不能再与中央地图台常驻同屏。
4. 中央地图台必须是页面视觉中心，且在移动端保持固定比例，不因旁边按钮收缩失衡。
5. 操作按钮需要比当前主按钮体系更紧凑：
   - 更低的高度
   - 更小的横向 padding
   - 更紧的间距
   - 保持可点按，不做过度微型化
6. 页面壳层不能吞掉现有编辑器交互：
   - 单指拖拽画路
   - 双指拖拽/缩放
   - 平移地图
   - 点击 finish 改起跑方向
7. 用户要求默认中文说明与现有视觉语义保持一致；本次不做无关视觉重构。

## Design Decisions

### 1. 移动端采用 A-compact 结构

- 采用“左工具窄栏 + 中央固定地图台 + 右状态窄栏”的三段式。
- 左侧仅保留高频编辑工具：
  - `画路`
  - `擦除`
  - `新建`
  - 一个进入赛道列表的次级入口
- 右侧仅保留与当前编辑对象直接相关的信息：
  - 赛道名称输入
  - 当前格子数/校验状态
  - 精简版操作提示
  - 保存按钮
- 这样做的原因：
  - 中央地图台是核心操作面，不应被上下栏挤压高度。
  - 左右窄栏比顶部/底部条更适合横屏设备，也更不容易和系统手势区冲突。

### 2. 移动端按钮使用“紧凑工具键”而不是通用大按钮

- 现有 `primary-action` / `secondary-action` 在移动端控制台里过大，适合作为普通产品 CTA，不适合工具栏。
- 本次为赛道编辑器引入局部紧凑样式，作用域限定在 track editor：
  - 工具按钮高度约 34px 到 38px
  - 文字尺寸略小于普通 CTA
  - 横向 padding 收窄
  - 按钮间距收紧
- 保存按钮仍然保留主行动语义，但在移动端也应压缩尺寸，避免成为过高的底部大块。
- 设计原则是“紧凑但可打”，而不是一味缩小。

### 3. 地图台固定比例且优先占面积

- `track-editor-stage-shell` 在移动端不再只是一个普通长方形内容块，而是控制台中心区域：
  - 使用固定或接近固定的宽高比
  - 与左右窄栏一起组成稳定的横屏骨架
  - 通过边框、内阴影或弱高光强化“操作台”感
- 中央地图台应尽量占用横屏一屏的主要面积；工具栏和状态栏围绕它而不是压缩它。
- 如果屏幕再小一档，优先继续缩小外围信息密度，而不是先压缩地图台可操作面积。

### 4. 已保存列表退出主同屏，改为次级面板

- 用户已明确要求“只保留编辑操作同屏”，因此已保存列表必须脱离主编辑区。
- 第一期实现可以使用最小改动方式：
  - 保留现有列表渲染逻辑
  - 但将其放到折叠区、抽屉、覆盖层或次级块中
  - 默认关闭，不抢占中央编辑空间
- 入口位置放在左侧工具区最下方最合适，因为它属于低频管理动作。
- 这样可以在不删除已有加载/编辑/删除逻辑的前提下，把页面结构调整到用户需要的“编辑优先”模式。

### 5. 操作提示压缩成短句状态提示

- 当前多条提示 chip 在移动端会明显挤压布局。
- 移动端只保留最必要、最短的提示，例如：
  - `单指画路 / 擦除`
  - `双指拖拽缩放`
  - `点终点切方向`
- 提示以紧凑文本块或 2-3 行短句呈现，而不是继续展开多枚横向胶囊。

## Component Responsibilities

### `src/components/TrackEditorClient.tsx`

- 继续负责：
  - 赛道名称状态
  - 当前 tool 状态
  - runtime mount / unmount
  - 保存、删除、加载、刷新赛道列表
- 新增职责：
  - 定义“主编辑控制台”和“已保存赛道次级面板”的分区结构
  - 提供显式的赛道列表开关状态
  - 在移动端主结构中把高频操作与低频赛道管理动作分离

### `src/app/globals.css`

- 继续负责全局与控制台页面的共享样式。
- 新增职责：
  - 为赛道编辑器增加移动端横屏专用断点样式
  - 提供 track editor 范围内的紧凑工具键样式
  - 提供中央地图台固定比例和窄栏尺寸约束
  - 提供默认隐藏的赛道列表次级面板样式

### `js/TrackEditorRuntime.js`

- 继续只负责 3D 编辑器运行时，不承担产品布局责任。
- 本次不修改交互模型；仅要求外部容器布局不能破坏其 pointer/touch 行为。

## Responsive Behavior

### Desktop / Large Tablet

- 保留现有“双列主编辑 + 已保存列表”的桌面阅读习惯是可接受的。
- 如果实现成本更低，也可以统一切成“主编辑 + 可展开列表”的结构，但不是本次核心要求。
- 本次重点是移动端，不要求桌面视觉大改。

### Mobile Landscape

- 进入 A-compact 模式：
  - 页面整体收口成一张紧凑控制台
  - 顶部标题区压缩
  - 主体三栏布局生效
  - 地图台固定比例并居中
  - 列表默认收起
- 所有核心编辑动作应在一个 viewport 内可见且无需纵向滚动。

### Mobile Portrait

- 不是本次主要目标，但不能完全破坏。
- 若竖屏空间不足，可允许比横屏略多的信息折叠，但仍应优先保证地图台和保存动作可达。
- 不需要为竖屏单独做一套复杂 UI，只需保持可用且不明显错位。

## Interaction Rules

1. 画路 / 擦除仍通过切换 `tool` 实现，不改变 runtime API。
2. 新建仍调用 `resetEditor()`，不改变清空逻辑。
3. 保存仍调用 `saveTrack()`，不改变后端交互和消息语义。
4. 赛道列表入口只负责打开次级面板，不触发数据结构变化。
5. 在次级面板中继续保留：
   - 编辑某张已保存赛道
   - 删除某张已保存赛道
6. 打开或关闭次级面板不能重建 runtime，也不能丢失当前未保存草稿。

## Testing Requirements

1. 组件测试至少验证：
   - 原生 runtime 仍然会挂载
   - 工具切换仍然触发 `setTool`
   - 赛道列表入口存在
   - 列表默认不是主编辑同屏的一部分
2. 手工验证至少覆盖：
   - 移动端横屏下一屏可见地图台、工具键、名称、状态、保存
   - 中央地图台可正常单指画路
   - 双指拖拽/缩放未被页面壳层拦截
   - 打开已保存列表后仍可编辑或删除已有赛道
   - 关闭列表后编辑器草稿保持不变
3. 回归验证：
   - `TrackEditorClient` 现有保存链路未破坏
   - 桌面端基本可用

## Acceptance Criteria

1. 移动端横屏进入自定义赛道页时，不滚动即可看到并操作核心编辑控件。
2. 中央地图台是视觉中心，并保持固定比例或稳定尺寸，不再被页面内容简单向下堆叠。
3. 工具按钮比现有大按钮明显更紧凑，且布局更集中。
4. 已保存赛道列表默认不与主编辑区常驻同屏。
5. 当前赛道可继续画路、擦除、新建和保存。
6. 地图仍支持拖拽平移和缩放，页面壳层不干扰 runtime 触摸交互。
7. 不引入赛道保存、删除、加载逻辑回归。
