# 房间页横屏密度优化设计

## 目标

在房间页保持现有赛车主视觉语言的前提下，重排横屏布局，解决以下问题：

- 右侧操作区留白过多，按钮显得过大。
- 左侧车手席卡片过高，单屏密度不足。
- 房间页缺少和比赛页一致的赛道预览信息。
- 颜色选择不应成为进入房间后的额外前置步骤。

## 交互结论

### 1. 房间配色策略

- 玩家进入房间时，系统自动分配当前可用的第一个车身颜色。
- 房主创建房间时同样自动获得第一个可用颜色。
- 颜色切换能力保留，但降级为“可改项”而不是“必选项”。
- 车手卡和房间主操作文案不再强调“未选赛车”，而直接展示当前颜色或准备状态。

### 2. 房间页主布局

- 采用双列布局。
- 左列承载房间主信息和车手席。
- 右列承载赛道预览、圈数控制和核心操作按钮。
- 顶部信息压成一条紧凑 headline，不再为说明文案保留大面积空区。

### 3. 左列结构

- 车手席头部右侧放一条紧凑颜色切换行：`车身颜色 + 小色块`。
- 车手席保持 2x2 发车位，但整体卡片高度、字体和间距同步下压。
- 已入场车手卡片保留角色、昵称、准备状态和颜色标识。
- 空位卡片保留，但文案缩短为等待提示，不额外占用说明区。

### 4. 右列结构

- 第一块：赛道预览，复用比赛页 minimap 的视觉语言。
- 第二块：比赛圈数，控制器压成单排小按钮。
- 第三块：核心操作，只保留 `准备 / 发车 / 退出房间`。
- 底部长说明文案保留为一行提示，在极窄横屏下允许隐藏。

## 视觉约束

- 延续现有深色赛车控制台风格，不引入新的设计系统。
- 保留 `panel-kicker`、`status-pill`、`driver-card`、`race-panel` 等已有视觉语义。
- 新增的小地图预览要明显继承比赛页 `race-minimap` 的轨迹线和起点点位风格。
- 颜色切换块以紧凑方块呈现，不再使用大面积颜色按钮。

## 代码边界

### 组件

- `RoomLobbyPanel.tsx` 负责新的布局和信息层级。
- `ColorPicker.tsx` 升级为支持紧凑色块模式。
- `LapTargetControl.tsx` 保持能力不变，但支持更紧凑的布局容器。

### 房间规则

- `realtime-worker/src/RoomCoordinator.ts` 负责自动分配默认颜色。
- 自动分配逻辑必须保证颜色唯一，不覆盖已存在玩家的手动选择。
- 玩家重连或重复 join 时保留已有颜色，不重新抢占。

## 验证标准

- 新玩家进入房间后立即拥有可用颜色，不需要先点选车身颜色。
- 房主在单人房间只需准备即可发车。
- 手机横屏首屏中可以同时看到：房间头部、4 个发车位、赛道预览、圈数和核心按钮。
- 右侧操作区不存在大面积空白，按钮尺寸明显小于当前实现。
